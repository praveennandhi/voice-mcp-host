# Milestone 0.1 — Windows + macOS dictation (Tier 1)

> **Audience:** the engineer (or coding agent) who will build this. This document is the contract — what to build, what *not* to build, and where to crib proven patterns from.

> **Prerequisite reading:** `../README.md` for the product vision and tier model.

---

## Scope

### In scope (must work for 0.1 to ship)

1. Tauri 2 app launches on **Windows 10/11** and **macOS 13+** (Apple Silicon + Intel).
2. Global hotkey toggles recording (press to start, press again to stop). Default: **F2** on Windows, **F5** on macOS (F-keys behave consistently across both — see "Hotkey defaults" below for why not Cmd-something).
3. Microphone capture at 16kHz mono via `cpal` on both platforms.
4. Local `whisper-rs` transcription on the captured audio.
5. Overlay window shows recording / transcribing / ready / error states. Non-focusable on both platforms.
6. Transcribed text pastes at the cursor in whatever app was focused *before* the hotkey was pressed. Uses **Ctrl+V** on Windows, **Cmd+V** on macOS.
7. Original clipboard contents restored after paste.
8. Settings window with a minimal config UI (hotkey, model selection, mic device, paste delays).
9. First-run model download with a visible progress indicator.
10. **macOS only:** first-run permission flow — clear UI prompting for Microphone and Accessibility permissions, with deep-links to System Settings if denied. Without Accessibility, paste silently fails — this must be detected and surfaced loudly.
11. Config persisted to `%APPDATA%\voice-mcp-host\config.json` (Windows) or `~/Library/Application Support/voice-mcp-host/config.json` (macOS).
12. JSONL event log at the platform's logs directory.
13. Distributables: unsigned NSIS installer (Windows) + unsigned `.dmg` (macOS). Gatekeeper bypass documented in `INSTALL.md`.

### Out of scope (defer to later milestones)

- LLM integration / `ask, ...` mode (→ 0.2)
- MCP host / `agent, ...` mode (→ 1.0)
- TTS / spoken replies
- Code signing / notarization (unsigned is fine for 0.1)
- Streaming partial transcripts
- Voice activity detection beyond simple level gate
- App-specific paste overrides
- Telemetry, crash reporting, auto-update
- Selection capture (`Ctrl+C` / `Cmd+C` round-trip)
- Linux (no current plan)

### Definition of done

A new user on either platform can:
1. Download the platform-appropriate installer (NSIS / DMG) from a GitHub release, install it. On macOS, follow the documented right-click → Open / `xattr` bypass for unsigned apps.
2. Launch the app — settings window opens, model not yet downloaded.
3. **macOS only:** grant Microphone and Accessibility permissions when prompted; the app re-checks and surfaces remaining permission gaps clearly.
4. Click "Download model," see progress, see "Ready."
5. Press the configured hotkey in any application:
   - Windows: Notepad, Chrome, VS Code, Slack
   - macOS: TextEdit, Chrome, VS Code, Slack
6. Speak for 3–30 seconds.
7. Press the hotkey again. Overlay shows "Transcribing…" then "Done."
8. Their transcribed text appears at the cursor in the original application.
9. Their original clipboard contents are still there (paste manually with Ctrl+V / Cmd+V to verify).
10. Do this 10 times across 4 different applications on each platform without a crash.

If all of that works reliably on both platforms, 0.1 is done.

## Audio pipeline (`src-tauri/src/audio.rs`)

### Capture spec

- **Crate:** `cpal`
- **Sample rate:** request 16000 Hz. If the device doesn't support it, capture at the device's preferred rate and resample to 16000.
- **Channels:** request mono. If only stereo is available, mix down to mono (`(L+R)/2`).
- **Sample format:** capture as `f32`. (Whisper consumes f32 PCM normalized to `[-1.0, 1.0]`.) If device delivers i16, convert: `sample as f32 / 32768.0`.
- **Buffer:** ring buffer or growable `Vec<f32>` owned by the recorder. Hard cap at `max_record_seconds * 16000` samples.

### API

```rust
pub struct AudioCapture { /* ... */ }

impl AudioCapture {
    pub fn new(device_id: Option<&str>) -> anyhow::Result<Self>;
    pub fn start(&mut self) -> anyhow::Result<()>;
    pub fn stop(&mut self) -> anyhow::Result<Vec<f32>>;  // returns full 16kHz mono f32 buffer
}
```

### Notes

- Don't VAD-trim in 0.1. Just hand the whole buffer to whisper-rs.
- Reject buffers shorter than `min_record_ms` (default 300ms) and surface a "too short" error to the overlay.
- **macOS:** the first `cpal::Stream::play()` triggers the TCC mic prompt. If the user denies, subsequent stream opens return `BuildStreamError`. Detect this and surface a permission error to the overlay with a link to System Settings → Privacy & Security → Microphone.

## ASR pipeline (`src-tauri/src/asr.rs`)

### Model

- **Default:** `ggml-base.en.bin` (~150MB). Fast enough on CPU for short utterances.
- **Optional:** `ggml-small.en.bin` (~500MB), `ggml-medium.en.bin` (~1.5GB) selectable in settings.
- **Source:** Hugging Face — `https://huggingface.co/ggerganov/whisper.cpp/resolve/main/<filename>`
- **Storage:** `%LOCALAPPDATA%\voice-mcp-host\models\<filename>`
- **Verify:** SHA-256 against a hardcoded map of known good hashes (publish hashes in `model_store.rs`).

### API

```rust
pub struct Transcriber { /* whisper-rs context */ }

impl Transcriber {
    pub fn load(model_path: &Path) -> anyhow::Result<Self>;
    pub fn transcribe(&self, audio_16k_mono_f32: &[f32], language: &str) -> anyhow::Result<String>;
}
```

### Settings

- Language defaults to `"en"`. Make it configurable but don't expose a fancy picker — just a text input. v1.
- Beam size, temperature, etc. — use whisper-rs defaults; do not expose.

## Hotkey & recording state machine

### State machine

```
IDLE ──[hotkey press]──> RECORDING
RECORDING ──[hotkey press]──> TRANSCRIBING
TRANSCRIBING ──[whisper returns]──> PASTING
PASTING ──[paste done]──> READY
READY ──[5s timeout]──> IDLE

Any state ──[error]──> ERROR
ERROR ──[7s timeout]──> IDLE
```

Each transition emits a Tauri event to the overlay window so the UI updates.

### Overlay payloads (Tauri event `app://overlay-state`)

| State | Title | Subtitle |
|---|---|---|
| `recording` | "Listening" | "Press F2 to stop" |
| `transcribing` | "Transcribing" | "Converting speech…" |
| `pasting` | "Pasting" | "" |
| `ready` | "Done" | "Pasted ✓" |
| `error` | "Error" | (error message) |

### Hotkey defaults

- **Windows:** `F2`
- **macOS:** `F5`
- **Why F-keys, not Cmd-something:** Cmd+Shift+Space is taken by Spotlight extensions, Cmd+Option+anything tends to clash with app shortcuts. F-keys are reliably free across both platforms (excluding gaming keyboards remapped to media controls — document the override path for that case).
- **Mode:** toggle (press to start, press to stop).
- **Config key:** `dictation.primary_hotkey` (string parsable by `tauri_plugin_global_shortcut::Shortcut::from_str`). Different defaults per-platform in `default_config.json`.
- See reference implementation: `C:\HermesApps\speaktype-tauri\src-tauri\src\hotkeys.rs` lines 6–26 (Windows pattern; the same tauri-plugin-global-shortcut API works on macOS but the user-visible string differs).

### Recorder ownership

Use a single global `Mutex<Option<AudioCapture>>` in `app_state`. The hotkey handler:
1. If `None` → create AudioCapture, start it, set state. **Call `window_target::capture_foreground()` HERE, before recording starts**, and store the result. Show overlay (`recording`).
2. If `Some` → stop it, take the audio buffer. Show overlay (`transcribing`). Spawn a worker thread that transcribes → calls paste pipeline with the stored target → emits final overlay state.

## Paste pipeline (the subtle part)

This is where 90% of dictation apps get it wrong. **Port the Windows path verbatim, hand-write the Mac path carefully.**

### Shared trait (`src-tauri/src/platform/mod.rs`)

```rust
pub trait ClipboardOps {
    fn get_text(&self) -> Result<String, String>;
    fn set_text(&self, text: &str) -> Result<(), String>;
    fn send_paste_shortcut(&self) -> Result<(), String>;   // Ctrl+V on Win, Cmd+V on Mac
}

pub trait WindowTargetOps {
    fn capture_foreground(&self) -> TargetWindow;
}
```

`insertion.rs` is the cross-platform orchestrator that owns the swap + sleep + paste + restore choreography. It uses the trait. Platform modules implement it.

### Step-by-step (platform-agnostic)

1. **At hotkey press (BEFORE recording starts):**
   - Call `platform::window_target().capture_foreground()`. Store the target.
   - Show overlay with `focus: false`. The overlay must NEVER take focus on either platform, or the paste will go into the overlay instead of the target app.

2. **At paste time (after transcription completes):**
   - `insertion::paste_text_with_delays(text, paste_delay_ms, restore_delay_ms)`.
   - Sequence: save original clipboard → set clipboard to transcript → sleep `paste_delay_ms` (default 100) → send paste shortcut → sleep `restore_delay_ms` (default 500) → restore original clipboard.
   - On any failure, leave the transcript on the clipboard (user can paste manually) and write it to `<config_dir>/last_transcript.txt`.

3. **Target restoration:** if the foreground window has changed since capture, in 0.1 we **do not** force focus back. Paste into whatever is foreground and log the discrepancy. `strict_target_window` is a 0.2+ knob on both platforms.

### macOS implementation (`platform/macos.rs`)

**No prior art in `speaktype-tauri/` for the Mac path. Design fresh, following this spec.**

#### Clipboard get/set — `NSPasteboard`

Use `objc2-app-kit::NSPasteboard::generalPasteboard()`:

```rust
// Pseudocode — actual code uses objc2 method_id! / msg_send! macros
fn get_text() -> Result<String, String> {
    let pb = NSPasteboard::generalPasteboard();
    pb.stringForType(NSPasteboardTypeString)
      .map(|nsstr| nsstr.to_string())
      .ok_or_else(|| "NSPasteboard empty or non-string".into())
}

fn set_text(text: &str) -> Result<(), String> {
    let pb = NSPasteboard::generalPasteboard();
    pb.clearContents();
    let ok = pb.setString_forType(NSString::from_str(text), NSPasteboardTypeString);
    if ok { Ok(()) } else { Err("NSPasteboard setString failed".into()) }
}
```

Note: `NSPasteboard` stores typed *items* and may hold image / file / RTF data alongside string data. For 0.1 we only save/restore the string form. Save/restore of multi-format clipboard data is a 0.2+ refinement (it's rare and worth deferring).

#### Synthetic Cmd+V — `CGEvent`

```rust
use core_graphics::event::{CGEvent, CGEventTapLocation, CGEventFlags, CGKeyCode};
use core_graphics::event_source::{CGEventSource, CGEventSourceStateID};

const KEY_V: CGKeyCode = 9;  // ANSI 'V' on Mac

fn send_cmd_v() -> Result<(), String> {
    let src = CGEventSource::new(CGEventSourceStateID::CombinedSessionState)
        .map_err(|_| "CGEventSource::new failed")?;
    let down = CGEvent::new_keyboard_event(src.clone(), KEY_V, true)
        .map_err(|_| "CGEvent down failed")?;
    down.set_flags(CGEventFlags::CGEventFlagCommand);
    down.post(CGEventTapLocation::HID);

    let up = CGEvent::new_keyboard_event(src, KEY_V, false)
        .map_err(|_| "CGEvent up failed")?;
    up.set_flags(CGEventFlags::CGEventFlagCommand);
    up.post(CGEventTapLocation::HID);
    Ok(())
}
```

> **The Accessibility footgun:** `CGEventPost` is silently dropped if the app is not in System Settings → Privacy & Security → Accessibility. The call succeeds, no error, nothing happens. This is the #1 failure mode for Mac dictation apps. **The permission flow (see below) must verify Accessibility status *before* claiming the paste worked.**

#### Foreground app capture — `NSWorkspace`

```rust
use objc2_app_kit::NSWorkspace;

fn capture_foreground() -> TargetWindow {
    let ws = NSWorkspace::sharedWorkspace();
    let app = ws.frontmostApplication();  // Option<NSRunningApplication>
    match app {
        Some(app) => TargetWindow {
            hwnd: None,
            pid: Some(app.processIdentifier() as u32),
            process_name: app.localizedName().map(|s| s.to_string()).unwrap_or_default(),
            title: String::new(),       // Per-window title needs AX; defer
            title_hash: String::new(),
        },
        None => TargetWindow::default(),
    }
}
```

For 0.1 the app name + PID is enough. Per-window title via `AXUIElementCopyAttributeValue(.focusedWindow)` is a 0.2 refinement.

### Default timings (in `default_config.json`)

```json
{
  "insertion": {
    "paste_method": "clipboard_paste",
    "restore_clipboard": true,
    "paste_delay_ms": 100,
    "restore_delay_ms": 500,
    "copy_to_clipboard_on_failure": true

}
```

These values are battle-tested on Windows in the reference codebase. **macOS timings need empirical QA** — Mac apps generally tolerate similar delays but a few (notably Slack and some Electron variants) want longer paste delays. Default 100/500ms is the starting point; record any Mac-specific tunings in the milestone retrospective.

## macOS permissions (new section — no Windows analog)

macOS dictation apps die at first contact with TCC. There's no negotiating with the OS here — handle it cleanly or users uninstall.

### Permissions required

| Permission | Why | What breaks without it |
|---|---|---|
| **Microphone** | `cpal` opens the input stream | Recording silently captures zero samples |
| **Accessibility** | `CGEventPost` for synthetic Cmd+V | Paste silently fails — no error, just nothing |

### Permission state machine

On every app launch (and re-checked on hotkey press):

```
state = check_mic_permission() x check_accessibility_permission()

if state == (granted, granted)      -> READY
if state == (denied, *)             -> show "Microphone needed" screen
if state == (*, denied)             -> show "Accessibility needed" screen
if state == (unknown, *)            -> trigger TCC prompt (open audio stream briefly)
if state == (*, unknown)            -> trigger TCC prompt via AXIsProcessTrustedWithOptions
```

### Detection APIs

- **Microphone:** `AVCaptureDevice.authorizationStatus(for: .audio)` — returns notDetermined / restricted / denied / authorized. From Rust, bridge through objc2.
- **Accessibility:** `AXIsProcessTrustedWithOptions([kAXTrustedCheckOptionPrompt: true])` — returns bool. Passing `true` for the prompt option opens a system alert with a "Open System Settings" button on first call; subsequent calls just return the current state. Use `accessibility-sys` crate for the bridge.

### First-run UX (macOS only)

Settings window's body when permissions are missing:

```
┌──────────────────────────────────────────────┐
│  voice-mcp-host needs permission to work     │
├──────────────────────────────────────────────┤
│                                              │
│   ✓  Microphone   (granted)                  │
│   ✗  Accessibility   [Grant…] [Open Settings]│
│                                              │
│  Why Accessibility? voice-mcp-host sends     │
│  Cmd+V to the focused app to paste your      │
│  dictation. Without this permission, macOS   │
│  silently blocks that — you'll see "Pasted"  │
│  but nothing happens.                        │
│                                              │
│  Re-check ↻                                  │
│                                              │
└──────────────────────────────────────────────┘
```

- "Grant…" triggers `AXIsProcessTrustedWithOptions(true)` which opens the system dialog.
- "Open Settings" deep-links to `x-apple.systempreferences:com.apple.preference.security?Privacy_Accessibility`.
- "Re-check ↻" re-runs the detection (since the user may grant in Settings and switch back).

### Paste failure detection

When the paste pipeline runs and Accessibility is missing, `CGEventPost` returns success even though nothing happened. To detect this:

1. Before sending Cmd+V, read the foreground app's clipboard via NSPasteboard.changeCount.
2. Send Cmd+V.
3. Wait `paste_delay_ms`.
4. The clipboard.changeCount won't change from a paste (paste is one-way), so we can't use that.
5. **Pragmatic detection:** check Accessibility trust state once per launch and once per hotkey press. If `AXIsProcessTrusted()` is false, refuse to attempt paste; show the permission screen instead.

This is good enough for 0.1. A more thorough "did the paste actually land" check requires AX inspection of the focused field's contents, which is a 1.0+ feature.

## Config file

- **Location (Windows):** `%APPDATA%\voice-mcp-host\config.json`
- **Location (macOS):** `~/Library/Application Support/voice-mcp-host/config.json`
- **Resolution:** use Tauri's `path::app_config_dir()` — it does the right thing per platform.
- **Format:** JSON with `schema_version: 1` at the top.
- **First-run:** copy `default_config.json` (bundled resource) to the per-user location if it doesn't exist.

### Schema (0.1)

Default config is platform-aware. Ship two `default_config.<platform>.json` files or one file with per-platform overrides resolved at load time.

```json
{
  "schema_version": 1,
  "dictation": {
    "primary_hotkey": "F2",            // F2 on Windows, F5 on macOS — resolved per-platform at first-run
    "language": "en",
    "min_record_ms": 300,
    "max_record_seconds": 120
  },
  "audio": {
    "input_device_id": null,
    "samplerate": 16000
  },
  "asr": {
    "model_name": "ggml-base.en.bin",
    "model_cache_dir": null            // null → platform default (see below)
  },
  "insertion": {
    "paste_delay_ms": 100,
    "restore_delay_ms": 500,
    "copy_to_clipboard_on_failure": true
  }
}
```

Model cache defaults:
- Windows: `%LOCALAPPDATA%\voice-mcp-host\models`
- macOS: `~/Library/Application Support/voice-mcp-host/models`

Reserve top-level keys `llm`, `mcp`, `tts` for future milestones — don't define them now, but write the loader so unknown keys are tolerated.

## First-run UX

### Both platforms

1. App launches. Settings window appears. No model on disk.
2. **macOS only:** check Microphone + Accessibility permissions. If either is missing, show the permission screen (see "macOS permissions" section) and gate model download behind it.
3. Settings panel shows a single big button: "Download base.en (~150 MB)" plus a progress bar slot.
4. User clicks → app downloads → verifies SHA-256 → moves into the platform-appropriate model cache directory.
5. Settings panel switches to its normal view. Status shows "Ready."
6. Hotkey is now armed.

### Permission corner cases

- **Windows mic:** generally granted by default for desktop apps. If `cpal` fails because of the OS-level "Allow apps to access your microphone" toggle, surface a clear error in the overlay with a link to `ms-settings:privacy-microphone`.
- **macOS mic:** TCC prompt appears on first `cpal::Stream::play()`. If denied, the app must say so loudly and re-trigger by deep-linking to System Settings.
- **macOS Accessibility:** see dedicated section. This must be handled or the app appears broken on first use.

## Logging

- **Location (Windows):** `%APPDATA%\voice-mcp-host\logs\events.jsonl`
- **Location (macOS):** `~/Library/Logs/voice-mcp-host/events.jsonl`
- **Rotation:** daily files (e.g. `events-2026-05-15.jsonl`).
- **Events to log** (each as one JSON line):
  - `app_started`
  - `hotkey_registered` (with the key string)
  - `hotkey_pressed`
  - `recording_started` (with target window context)
  - `recording_stopped` (with duration_ms, sample_count)
  - `transcription_started`
  - `transcription_completed` (with chars, duration_ms; **no transcript text by default** — controlled by `privacy.verbose_transcript_logging`)
  - `paste_attempted`
  - `paste_completed` (with `PasteReport`)
  - `error` (with module, message)
- **PII:** never log transcript content unless `verbose_transcript_logging` is true.

## Open questions (decide before / during build, not blockers now)

1. **Push-to-talk vs toggle as default?** This doc specifies toggle. If the engineer building 0.1 prefers push-to-talk after trying both, that's a fine swap — but pick one and stick to it. Don't ship both modes simultaneously in 0.1.
2. **GPU acceleration?** whisper-rs may or may not expose Metal (Mac) / CUDA (Win) cleanly when 0.1 is built. CPU-only is acceptable for 0.1 ship; GPU paths can be 0.2 polish. If only one platform's GPU works at ship time, enable that one and document.
3. **Final app name?** Folder/binary still uses `voice-mcp-host`. Rename before public 0.1 release. Not blocking dev work.
4. **Tray icon behavior?** `tauri-feature: ["tray-icon"]` is enabled in deps but 0.1 spec doesn't define tray UX. Either ship a minimal tray (right-click → Settings / Quit) or hide it. Engineer's call; document the choice in the PR.
5. **Mac per-window title capture?** Capturing the focused window's *title* on macOS requires the Accessibility API (`AXUIElementCopyAttributeValue`). 0.1 captures only the app name + PID. If logs become hard to debug without titles, consider promoting this to 0.1 scope; otherwise defer to 0.2.
6. **Intel Mac support?** GitHub Actions can build for both arm64 and x86_64. Building both doubles CI time. Decision: ship arm64 only at 0.1.0, add Intel at 0.1.1 if anyone asks. Document as a known limitation.
